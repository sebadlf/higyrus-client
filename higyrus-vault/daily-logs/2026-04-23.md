## Aprendizajes

- **Linear↔GitHub integration tenía scope al repo viejo**: al crear issues en el project `Higyrus Client` (team `BEC`), Linear abrió GitHub issues fantasma en `sebadlf/matriz-client` (#97–#101). Fix: reconfigurar la integración con `sebadlf/higyrus-client` en Team settings. Vale documentar esto como runbook la próxima vez que un team de Linear sirva a varios repos.
- **`enablePullRequestAutoMerge` hay que habilitarlo en settings del repo** (`Settings → General → Pull Requests → Allow auto-merge`). El workflow `auto-merge.yml` falla con `GraphQL: Auto merge is not allowed for this repository` hasta que se habilita. No es parte de Branch protection — es un toggle aparte.
- **Pydantic no era la mejor opción** — validación estricta pelea con el modelo de keys condicionales de Higyrus. Frozen dataclasses + `from_api` resultaron más simples, sin dependencias, y alcanzan el objetivo (chaining seguro). Ver ADR-001.
- **La foundation valió lo que costó**: en BEC-78 el wrapper `get_posiciones` terminó siendo ~15 líneas efectivas. Todo el trabajo de parsing, defaults, serialización de fecha/bool y drop de `None` ya estaba hecho en BEC-80 y BEC-81. Los próximos endpoints de lectura deberían ser igual de chicos.
- **Convención adoptada**: tests por dominio (`test_cuentas.py`) en vez de por archivo Python (`test_client.py`). `test_client.py` queda para tests del transporte (`_request`, `_raise_for_response`, login); los endpoints públicos agrupados por sistema Higyrus (`cuentas`, `personas`, `operaciones`, etc.).
- **Tri-state bools en wrappers**: cuando un bool tiene "no especificado" como estado válido en la API (p. ej. `ocultarCerradas` en `/posicionValuada`), la signatura es `bool | None = None` en vez de `bool = False`. Así `None` → dropeado y no se manda el param (server usa su default); `True`/`False` → se manda explícito. Distinto a casos como `incluir_parking: bool = False` donde siempre enviamos el flag para debugging explícito.
- **Keys del API con acentos**: el PDF muestra algunas keys de response con acentos castellanos (`información`, `fechaCotización`, `valuación`, `sesión` en `/posicionValuada`). El resto del API no usa acentos. Asumimos artifact de OCR y modelamos sin acentos. Si sandbox demuestra lo contrario, agregamos aliases en `from_api` — **no renombrar fields**, porque la convención del vault es mirror 1:1 del wire format.
